# 文档切片策略说明

## 什么是文档切片？

RAG（检索增强生成）系统中，上传的文档需要被切分成多个小片段（chunk）才能进行向量化存储和检索。切片策略决定了如何切分文档，直接影响检索精度和回答质量。

---

## 四种切片策略

### 1. 字符滑动窗口（fixed_size）— 默认

**原理**：按固定字符数切分，相邻 chunk 之间有重叠部分。

```
文档内容: [------100字------][------100字------][------100字------]
                     重叠20字↑↑↑↑↑↑↑↑
```

**参数**：
- `chunk_size=1024`：每个 chunk 的最大字符数
- `chunk_overlap=100`：相邻 chunk 之间的重叠字符数

**优点**：
- 实现简单，性能最好
- 任何文档类型都能处理
- 重叠部分防止信息在边界处断裂

**缺点**：
- 可能切断句子或段落，破坏语义完整性
- chunk 边界是任意的，可能将相关信息分到不同 chunk

**适用场景**：通用场景、对性能要求高的场景、文档没有明显结构时使用

---

### 2. 语义边界分块（sentence）

**原理**：以句子为基本单位切分，保证每个 chunk 边界都在句号、问号、感叹号等句子结束位置。

```
文档内容: [第一句话。第二句话。第三句话。][第四句话。第五句话。]
          ↑边界在句号处          ↑边界在句号处
```

**支持的中英文分隔符**：`。！？.!?\n`

**优点**：
- 每个 chunk 语义完整，不会切断句子
- 对中文文档友好，符合自然阅读习惯
- 检索到的片段通常是完整的表述

**缺点**：
- 长句子可能导致 chunk 超出预设大小
- 对没有标点的文本（如代码、表格）效果不佳

**适用场景**：中文文档、小说、文章、报告等自然语言文本

---

### 3. Markdown 标题分块（markdown_header）

**原理**：按 Markdown 标题（`##`、`###`）位置切分，每个标题及其下方内容作为一个独立 chunk。

```
文档内容:
[## 产品概述                          ]
[  产品是一款智能助手平台...          ]
[## 核心功能                          ]
[### 文档解析                        ]
[  支持PDF、Word等格式...            ]
```

**优点**：
- 严格按文档结构切分，语义完整性最好
- 每个 chunk 自带标题上下文
- 适合结构化文档

**缺点**：
- 仅适用于 Markdown 格式文档
- 无标题的文档会退化为一个整体

**适用场景**：MD 文档、技术文档、README、有良好结构的文档

---

### 4. 递归分块（recursive）— 推荐

**原理**：分层降级策略，优先保持语义完整性，遇到超长文本自动降级到更细粒度。

```
输入文本
    ↓
按段落(\n\n)切分 ──→ 每段 ≤ chunk_size ──→ ✅ 保留
    ↓ 段落太大
按句子(。！？.!?)切分 → 每句 ≤ chunk_size ──→ ✅ 保留
    ↓ 句子太大
按字符滑动窗口切分 ──→ 强制切分到 chunk_size
```

**三段降级逻辑**：
- Level 1 - 段落切分（`\n\n`）：保留段落完整性
- Level 2 - 句子切分（`。！？.!?`）：段落太大时降级到句子
- Level 3 - 字符滑动窗口：单个句子超长时强制按字符切分

**优点**：
- 综合了以上三种策略的优点
- 尽可能保持语义完整性
- 极端情况有兜底，不会出现超长 chunk
- 混合文档（既有段落又有长表格）表现最佳

**缺点**：
- 实现相对复杂
- 不如 fixed_size 性能高

**适用场景**：混合类型文档、未知格式文档、推荐作为默认策略

---

## 策略对比总结

| 策略 | 语义完整性 | 性能 | 兜底机制 | 推荐度 |
|------|-----------|------|---------|-------|
| 字符滑动窗口 | ❌ 差 | ⚡ 最快 | ✅ 有 | ⭐⭐ |
| 语义边界 | ✅ 中 | ⚡ 快 | ❌ 无 | ⭐⭐⭐ |
| Markdown标题 | ✅✅ 最好 | ⚡ 快 | ❌ 无 | ⭐⭐⭐ |
| 递归分块 | ✅✅ 好 | ⚡ 中 | ✅ 完善 | ⭐⭐⭐⭐⭐ |

## 如何配置

### 通过 .env 文件

```ini
# 默认切片策略
chunk_strategy=sentence        # 可选: fixed_size / sentence / markdown_header / recursive
chunk_size=1024                # 每块最大字符数
chunk_overlap=100              # 相邻块重叠字符数
```

### 通过前端上传页面

在文档管理页面的下拉框中选择策略后上传文档，选择会覆盖 `.env` 中的默认值。

---

## 技术实现

所有切片策略实现在 `src/parsing/chunker.py` 中，通过 `Chunker` 类统一调用：

```python
from src.parsing.chunker import Chunker

chunker = Chunker(
    strategy="recursive",      # 策略名称
    chunk_size=1024,           # 每块最大大小
    chunk_overlap=100,         # 重叠大小
)
chunks = chunker.chunk(parsed_docs)
```

每种策略对应一个内部方法：
- `fixed_size` → `_fixed_size_split()`
- `sentence` → `_sentence_split()`
- `markdown_header` → `_markdown_header_split()`
- `recursive` → `_recursive_split()`
