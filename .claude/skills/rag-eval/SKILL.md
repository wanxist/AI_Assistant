---
name: rag-eval
description: Evaluate RAG retrieval quality — precision, recall, hallucination, and per-document scoring.
---

当用户调用 `/rag-eval` 时，执行以下流程：

## 1. 确认范围
询问用户：
- 评测哪些文档？（默认：data/documents/ 下最近上传的 5 个文件）
- 是否需要和用户手动标注的正确答案对比？（默认：不需要，用 LLM 自动判断）

## 2. 生成测试问题
对每个文档，生成 3 类问题（每类 2 个，共 6 个/文档）：
- **事实查询**：文档中明确记载的信息（如 "XX 的截止日期是什么？"）
- **推理题**：需要综合多处信息才能回答（如 "A 和 B 方案哪个更经济？"）
- **边界题**：文档中未提及或模糊的信息（如 "文档没有提到的风险有哪些？"）

## 3. 执行评测
对每个问题调用 `POST /query {"question": "...", "top_k": 5}`，记录：
- `answer`：LLM 回答
- `sources`：引用的来源列表
- `score`：每个 source 的相关性分

## 4. 评分维度

| 维度 | 标准 |
|------|------|
| 召回率 | 回答中引用的文档 chunk 是否覆盖了正确答案所需信息 |
| 精确率 | 引用的 sources 中，真正相关的占比 |
| 幻觉率 | answer 中包含文档中不存在的信息的比例 |
| 拒答率 | 边界题中正确返回 "未找到相关信息" 的比例 |

## 5. 输出报告
```
| 文档 | 事实题 | 推理题 | 边界题 | 综合 |
|------|--------|--------|--------|------|
| doc1 | 4/5    | 3/5    | 5/5    | 80%  |
| doc2 | ...    | ...    | ...    | ...  |

总结：
- 平均精确率：xx%
- 平均召回率：xx%
- 幻觉率：xx%
- 建议：...
```
